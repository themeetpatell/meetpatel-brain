# Founder Library — Map of Content

> 20 founder / operator / strategy books distilled and made Meet-applicable. Each note: why it matters → core frameworks → killer quotes → how to apply to ventures → related. This is the canonical first wave. More to come.

## How to use this library

Each note is a **standing reference**, not a one-time read. Open the note when the situation it describes lands:
- Hiring senior? → *Multipliers*
- A hard conversation looming? → *Radical Candor*
- A wedge needs choosing? → *Crossing the Chasm*
- Need to write a PR/FAQ for a launch? → *Working Backwards*
- Whirlwind drowning the strategic priority? → *Amp It Up*

The notes are NOT replacements for the books. They are **forcing functions for application** — every note ends with "how I'd apply to [venture]" to make the IP actionable.

## Cross-book themes (worth noticing)

- **Brutal honesty as competitive advantage** — Horowitz, Slootman, Scott, Walsh, Duke all converge on this. Founder-friend coaching books usually don't.
- **Cadence beats brilliance** — Grove, Wickman, Slootman, Walsh. Operating systems compound.
- **Customer language beats internal language** — Cagan, Fitzpatrick, Dunford, Heath. Positioning is built from listening, not whiteboard.
- **Asymmetric bets, not balanced portfolios** — Thiel, Helmer, Duke. Variance is the friend; covariance is the enemy.
- **The 1:1 is the highest-leverage meeting on earth** — Grove, Scott, Wiseman. Every operating book agrees.

## How to add a new book

1. Create note in the right category subfolder
2. Use this standard structure:
   - **Why it matters to Meet** (1 paragraph)
   - **One-line thesis** (the book in one sentence)
   - **Core ideas / frameworks** (3–5 max)
   - **Killer quotes** (3–5)
   - **How to apply to [venture]** (at least one concrete example)
   - **Anti-patterns / where the book is wrong**
   - **Related notes**
   - **Source / read this next**
3. Wikilink from this MOC
4. Tag: `book` + category

## Maintenance

- Quarterly: review which books got opened, which didn't. Curate the library based on actual use.
- After every venture decision, retro: which book in this library would have helped? If none, add the missing one.
