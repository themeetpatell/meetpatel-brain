# Growth Loops (vs Funnels)

> Funnels are linear: pay → acquire → churn. Loops are circular: each user's action produces the next user's acquisition. Funnels need fuel poured in. Loops compound. Loops are why durable companies grow without bleeding cash.

## When to use it
- Designing a new growth strategy from scratch
- When paid acquisition is the only thing working (red flag — no loop)
- When CAC is creeping up (signal: funnel-mode, not loop-mode)
- When investors ask "what's your defensible growth motion?"

## The framework

A loop has 4 components:

1. **Input** — what the user does (registers, posts, shares, files something)
2. **Action** — the system's response that creates artifact (a profile, a piece of content, a published file)
3. **Output** — the new user, view, or interaction that artifact generates
4. **Reinvestment** — the output loops back as a new input

The classic shapes:
- **Viral loop:** user → invites/shares → new user
- **Content loop:** user creates content → indexed/discoverable → new user discovers → joins
- **Sales loop:** customer success → case study / referral → new customer
- **Paid loop:** revenue → reinvested in ads → more revenue (counts only if LTV > 2× CAC)

A real growth motion has **multiple loops compounding**, not one.

## How to apply

### Finanshels — loops audit

- **Sales loop (current strongest):** WhatsApp inbound → conversion → customer becomes channel partner → referrals → new WhatsApp inbound. **Active. Reinforce.**
- **Content loop (under-built):** Meet's public posts on Finanshels CoE → readers contact us → become customers → produce more public moments. **Build via personal brand engine.**
- **Product loop (weak):** customer uses dashboard → dashboard surfaces value → customer expands services. **Build via dashboard polish + automated upsell triggers.**
- **Paid loop:** ads → conversion → revenue → reinvest. **Only works if LTV:CAC stays >3×.** Track monthly.

### Biggdate — loops audit (mostly aspirational pre-launch)

- **Viral loop:** "you can't really invite people to a dating app, but you CAN signal you're on it" — branded swag, dignity-aesthetic posts, founder-led narrative
- **Content loop:** stories of real outcomes → press / social → new sign-ups
- **Network loop:** more verified members → better matches → higher word-of-mouth → more members

Network effects in dating are real but slow and city-specific. Plan for that.

### StartupOS — loops audit (pre-launch)

- **Product-led loop:** founders use it → produce visible artifacts (decision logs, OKR docs) → other founders ask "how" → conversion
- **Content loop:** Meet writes about building it → readers want it → users
- **Community loop:** users share decisions / templates → community library grows → more users

## Anti-patterns
- **Calling a funnel a loop.** A funnel that ends at "customer" is not a loop. The output has to *re-enter* as input.
- **Building one loop and stopping.** Real companies have 3+ compounding loops.
- **Optimizing the wrong loop.** If your strongest loop is paid, that's not durability — that's just a cost-of-customer pipeline.
- **Treating viral coefficient like a hack.** k > 1 sustainably requires real value, not gimmicks. Most "growth hacks" don't loop.
