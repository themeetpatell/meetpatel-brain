# Hooked Model

> Habit-forming products run through a 4-step loop: Trigger → Action → Variable Reward → Investment. Each pass through the loop deepens the user's attachment. The model is powerful — which makes it ethically loaded. Use deliberately.

## When to use it
- Designing onboarding for retention
- Debugging: "users sign up but don't come back"
- Reviewing notification strategy and re-engagement
- Considering the *ethics* of engagement features (use the framework to also check for harm)

## The framework — 4 phases

1. **Trigger** — what brings the user back?
   - *External:* notification, email, social cue, ad
   - *Internal:* an emotion or state (boredom, loneliness, anxiety, FOMO) that the user has learned to associate with your product
   - Internal triggers are the goal. External triggers stop working.

2. **Action** — the simplest behavior in anticipation of reward
   - Open the app. Scroll. Tap. Type. The action should be near-zero friction.

3. **Variable Reward** — *unpredictable* payoff
   - Three categories: (a) Reward of the tribe (social validation), (b) Reward of the hunt (resources/info), (c) Reward of the self (mastery/completion)
   - Variability matters — if reward is predictable, the loop fades

4. **Investment** — user does something that increases future value
   - Adds data, follows, builds reputation, creates content, completes profile
   - The investment makes the *next* trigger more compelling

## How to apply

### Biggdate (use with caution — dating app ethics matter)
- **Trigger:** Daily curated match drop (8am UAE, predictable timing for daily ritual) — internal trigger develops as "morning coffee + my Biggdate match"
- **Action:** Open app, see today's profile
- **Variable Reward:** *Quality* of match varies. Reward of the tribe (social affirmation in match quality), reward of the hunt (potential partner)
- **Investment:** Profile depth, conversation history, intent signals — each interaction makes future matches better

*Ethics check:* Don't optimize for "swipes per session". Optimize for outcomes (real connections leading to off-app dates). If the loop creates compulsive use without proportional benefit, you've built Tinder, not Biggdate.

### StartupOS
- **Trigger:** Monday morning "what's your top priority this week?" prompt (external → becomes internal habit)
- **Action:** Open OS, set the priority
- **Variable Reward:** AI surfaces context, suggests prior decisions that apply, drafts the first move
- **Investment:** Every decision logged makes future decisions smarter — compounding value

### Finanshels
- **Trigger:** Filing deadline approaching (calendar-driven, partly external)
- **Action:** Open dashboard, review what's been auto-filed
- **Variable Reward:** "Status: filed, you're good for the month" + occasional surprise insights ("you can claim X additional deduction")
- **Investment:** Each filing adds to history, makes future filings faster, creates compliance trail

## Ethics layer (mandatory check)

For every Hooked design:
- Does the user *want* this habit if they think about it clearly?
- Does the loop benefit them or just us?
- Are the variable rewards substantive or manipulative?
- Is the investment compounding *for them* or just *locking them in*?

The Hooked model is real and works. Misused, it builds slot machines. Used well, it builds tools people are glad they kept using.

## Anti-patterns
- **All external triggers, no internal trigger formation.** The loop fades when you stop pushing notifications.
- **Fixed rewards instead of variable.** Predictable reward = habit decays.
- **Investment that's all extraction (data hoarding) and no value-back.** Loop dies when user notices.
- **Engagement loops without value creation.** Tells the user they're being farmed.
