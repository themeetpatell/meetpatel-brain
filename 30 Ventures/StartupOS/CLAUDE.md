# StartupOS — Agent Context

> Read this before doing any work on StartupOS. This is the canonical agent-readable brief.

## One-paragraph snapshot

StartupOS is an **AI-native operating system for founders** — copilots, execution systems, ecosystem access, intelligence. The highest-leverage expression of Meet's broader thesis that *operating infrastructure is the next venture-scale opportunity in the founder market*. Currently in build / wedge-selection mode. Pre-seed stage when the narrative locks. 805 legacy files (from earlier ideation cycles) need collapsing into a sharp working layer.

## Current state (refresh quarterly)

- Stage: build / wedge selection
- Team: small / founder-led, hires to come post-wedge-lock

- Status: 805 legacy files exploring multiple feature directions — need radical compression to 25 working notes

## Wedge candidates (current thinking)

1. **AI Chief of Staff for solo founders** — narrowest, fastest, lowest-risk
2. **AI-native execution OS for early-stage teams** (5–20 people) — adjacent to Linear / Notion / ClickUp but with native intelligence
3. **Founder ecosystem layer** — community + mentorship + AI matching (more network-defensible, slower to monetize)
4. **AI-native fundraise OS** — narrow vertical inside founder workflow

Decision needed: which wedge ships first. Don't ship all four.

## ICP (current hypothesis — to validate)

- First-time + second-time founders, 0–10 employees, $0–500k ARR
- Bias toward technical founders comfortable with AI tooling
- Geography: global but start with Dubai/MENA + Indian diaspora founders for distribution advantage
- Pain triggers: feeling under-resourced, no co-founder for strategic thinking, drowning in tooling, slow execution velocity

## Current strategic priorities

1. **Lock the wedge** — pick 1 of 4 candidate wedges, kill the others, write the decision frame
2. **Collapse legacy** — 805 files → 25 working notes
3. **Build minimum proof** — shippable demo of the wedge, real users
4. **Narrative lock for pre-seed** — investor-ready one-pager + 90-day proof plan

## Voice + brand

- **Operator-grade, technical, slightly contrarian.** Not "founder-friendly" in the soft sense.
- Speaks to people who've shipped something. Doesn't speak down.
- AI is the substrate, not the marketing line. Don't say "AI-powered". Show what the AI does.
- Brand neighbors in tone: Linear, Cursor, Granola, Notion (early). Not WeWork / Mindvalley energy.

## What NOT to do on StartupOS work

- Don't expand scope. Every "and also" kills the wedge.
- Don't write generic startup advice copy. It's the easiest way to sound like a Twitter coach.
- Don't ship a "platform" pitch. Ship a tool that does one thing well.
- Don't fund-raise on vision alone — proof anchors the narrative.
